如何在本地部署DeepSeek-R1蒸馏模型？

技术教程 • 2025年2月8日

如何在本地部署DeepSeek-R1蒸馏模型？——以Ollama框架为例

本文章以windows系统为基础写的教程，其他系统用户请移步。

作为一名技术爱好者，今天我来分享一下如何在本地部署DeepSeek-R1蒸馏模型。本地部署deepseek-r1我推荐使用ollama框架，这篇文章我也仅给出ollama+deepseek这一个部署方案，下面就是我的实战经验分享，希望能帮到大家。看不懂操作过程的可以移步到结尾看视频 :P

一、为什么选择Ollama？

在尝试过几种不同的模型部署方案后，我认为Ollama框架有以下几个明显优势：

  • 轻量级设计：占用资源少，适合普通PC部署
  • 支持多种主流模型：包括DeepSeek-R1在内的众多开源模型
  • 界面简洁易用：命令行操作简单直观
  • 社区活跃：有丰富的文档和教程资源

二、部署准备

在开始之前，请确保你的电脑满足以下基本条件：

操作系统：Windows 10/11（本文以Windows为例）

硬件要求：

  • CPU：支持AVX2指令集的Intel/AMD处理器
  • 内存：至少16GB RAM（根据模型大小可适当增加）
  • 显卡：NVIDIA显卡 + CUDA支持（非必选项，但能显著提升性能）

三、详细部署步骤

1.安装Ollama框架

首先我们需要安装Ollama框架。访问Ollama官方网站（https://ollama.ai/），找到适合Windows系统的安装包进行下载。

安装过程非常简单，一路点击"下一步"即可完成。安装完成后，我们可以通过以下方式验证是否安装成功：

按Win键+R，输入cmd回车，然后在命令行中输入：

ollama

如果显示类似下方的界面，则说明安装成功：

Usage: ollama [command]

Available Commands:
    api      Start the API server 
    build    Build a model from a recipe 
    chat     Chat with a model 
    ...

2.配置存储路径

由于默认情况下Ollama会将模型文件存储在C盘系统目录下，而C盘通常空间有限，因此建议我们将存储路径指向其他磁盘分区。

具体操作步骤如下：

  • 右键点击任务栏，选择"搜索"
  • 在搜索框中输入"环境变量"
  • 点击"编辑系统环境变量"
  • 在"系统变量"部分找到"Path"变量
  • 新建一个名为"OLLAMA_MODELS"的变量
  • 设置值为你希望存储模型的位置（例如D:\ollama_models）

3.下载DeepSeek-R1模型

现在我们可以开始下载DeepSeek-R1模型了。以下是具体步骤：

a) 打开命令行工具（cmd）

b) 输入以下命令搜索可用模型：

ollama search deepseek-r1

c) 选择合适的模型版本。目前Ollama提供了多个不同大小的DeepSeek-R1模型版本：

  • 1.5B参数量：适合显存较小的设备（最低4GB显存）
  • 7B参数量：适合中端显卡（推荐8GB显存）
  • 32B参数量：需要较大显存支持（建议24GB以上）
  • 671B参数量：最高规格版本，需要充足的硬件资源

d) 下载模型。以下载7B版本为例，执行命令：

ollama pull deepseek-r1:7b

4.启动模型服务

模型下载完成后，就可以启动模型服务了。执行以下命令：

ollama serve

这将启动一个本地API服务，默认监听在http://localhost:11434端口

5.测试模型

现在我们可以测试一下模型是否正常工作了。执行以下命令：

ollama chat --model deepseek-r1:7b

然后你就可以和DeepSeek-R1进行对话了！

四、注意事项

硬件要求：

  • 1.5B版本：最低4GB显存
  • 7B版本：推荐8GB显存
  • 32B版本：建议24GB显存
  • 671B版本：需要充足硬件资源支持

存储空间：

  • 1.5B版本：约40GB
  • 7B版本：约200GB
  • 32B版本：约800GB
  • 671B版本：约4TB

性能优化建议：

  • 如果你的显卡支持CUDA加速，可以在启动服务时添加--gpu参数
  • 可以通过设置环境变量来调整显存分配比例

五、总结

通过以上步骤，我们就可以在本地成功部署DeepSeek-R1蒸馏模型了。这种方式相比云端部署具有以下优势：

  • 成本更低：无需持续支付云服务费用
  • 响应更快：本地部署延迟更低
  • 更加灵活：可以根据需要调整配置
